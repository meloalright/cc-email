# cc-email

[![CI](https://github.com/meloalright/cc-email/actions/workflows/ci.yml/badge.svg)](https://github.com/meloalright/cc-email/actions/workflows/ci.yml) [![homebrew](https://img.shields.io/badge/dynamic/regex?url=https%3A%2F%2Fraw.githubusercontent.com%2Fmeloalright%2Fhomebrew-tap%2Frefs%2Fheads%2Fmain%2FFormula%2Fcc-email.rb&search=version%20%22(%3F%3Cversion%3E%5Cd%2B(%3F%3A%5C.%5Cd%2B)%2B)%22&replace=%24%3Cversion%3E&label=homebrew-tap&color=f9d094)](https://github.com/meloalright/homebrew-tap/tree/main/Formula/cc-email.rb) [![npm](https://img.shields.io/npm/v/cc-email?color=red)](https://www.npmjs.com/package/cc-email) [![License: MIT](https://img.shields.io/badge/license-MIT-blue)](LICENSE)

A bridge between email and AI coding agents. Send a task from email, get the result back.

**No public IP. No webhook server. No self-hosted SMTP.**

##### ⚡️ install via homebrew

```sh
brew install meloalright/tap/cc-email
```

##### ⚡️ install via npm

```sh
npm install -g cc-email
```

## Quick start

### 1. Gmail App Password

1. Enable 2FA on your Google account
2. Create an App Password at https://myaccount.google.com/apppasswords
3. Export it:

```bash
export CC_EMAIL_GMAIL_PASSWORD="xxxx xxxx xxxx xxxx"
```

### 2. Configure

```toml
# cc-email.toml

[inbox]
type = "imap"
host = "imap.gmail.com"
port = 993
username = "you@gmail.com"
password_env = "CC_EMAIL_GMAIL_PASSWORD"
folder = "INBOX"
poll_interval_seconds = 15

[outbox]
type = "smtp"
host = "smtp.gmail.com"
port = 587
username = "you@gmail.com"
password_env = "CC_EMAIL_GMAIL_PASSWORD"
from = "you@gmail.com"

[agent]
type = "claude-code"
command = "claude"
timeout_seconds = 7200
permission_mode = "email"
permission_timeout_seconds = 7200
permission_default = "deny"

[security]
allowed_senders = ["your-email@example.com"]
max_body_bytes = 20000
max_attachment_bytes = 5000000
```

### 3. Run

```bash
cc-email listen --config cc-email.toml
```

Send an email to yourself with a coding task — Claude Code will process it and reply with the result.

## Features

- **Claude Code stream-json IPC** — native integration, not just a CLI wrapper
- **Permission approval via email** — when Claude needs to run a tool, you get an email asking to approve or deny
- **Attachment send-back** — files generated by Claude are automatically attached to the reply
- **Session management** — per-sender conversation sessions with history
- **Task persistence** — SQLite-backed task tracking
- **Security** — sender allowlist, body validation

## Commands

Send these as the email subject or body:

| Command | Description |
|---------|-------------|
| `/new` | Start a new session |
| `/doctor` | Run diagnostics |
| `/help` | List all commands |

More commands are planned for future releases.

## Other Email Providers (WIP)

cc-email works with any IMAP + SMTP provider. The following are planned but not yet fully tested:

| Provider | IMAP Host | SMTP Host | Auth |
|----------|-----------|-----------|------|
| Outlook | outlook.office365.com | smtp.office365.com | App Password / OAuth |
| Yahoo | imap.mail.yahoo.com | smtp.mail.yahoo.com | App Password |
| iCloud | imap.mail.me.com | smtp.mail.me.com | App-specific password |
| 163 Mail | imap.163.com | smtp.163.com | Authorization code |
| QQ Mail | imap.qq.com | smtp.qq.com | Authorization code |

## Other Agents (WIP)

Currently only Claude Code is supported as the agent backend. Custom command agents and other coding agents are planned for future releases.

## Architecture

```
src/
├── main.rs                         # CLI entry point
├── engine.rs                       # Engine orchestrator, in-flight task mgmt
├── config.rs                       # TOML config parsing
├── security.rs                     # Sender allowlist, body validation
├── permission.rs                   # Email-based permission approval flow
├── attachment.rs                   # File scanning, security filtering
├── inbox/
│   └── imap_poll.rs                # IMAP polling
├── mail/
│   ├── parser.rs                   # MIME email parsing
│   ├── reply.rs                    # SMTP reply with attachments
│   └── formatter.rs               # Reply body formatting
├── agent/
│   ├── command_runner.rs           # Generic subprocess agent
│   └── claude_code.rs              # Claude Code stream-json IPC
├── session/
│   └── mod.rs                      # Session management, history
├── task/
│   ├── model.rs                    # Task model
│   └── store.rs                    # SQLite persistence
└── command/
    └── builtins.rs                 # Built-in command handlers
```
